## 1. 用户场景地图

### 1.1 用户角色

| 角色 | 核心目标 | 使用频率 | 信任基线 |
|------|---------|---------|---------|
| 高频桌面工作者（单人，自用） | 工作中零摩擦记下闪过的待办，随手瞄/勾 | 每天数十次（碎片化、贯穿全天） | N/A（非 AI，无信任问题） |

### 1.2 用户日常旅程时间线

| 时段 | 用户正在做什么 | 核心需求 | 对应场景 |
|------|-------------|---------|---------|
| 工作中任意时刻 | 写代码/写文档/开会，念头闪过 | **不打断当前事**，1 秒记下 | S1 热键捕获 |
| 工作间隙 | 想确认"还有啥没做" | 不开 app 瞄一眼 | S2 hover 瞄看 / 静止数字 |
| 完成一件事 | 勾掉它 | 愉悦地划掉、清单变轻 | S3 勾选完成 |
| 整理片刻 | 调一下顺序/补细节 | 轻量编辑 | S4 编辑/排序 |
| 全部做完 | 看到空清单 | 友好的"清零"反馈 | S5 空态 |

### 1.3 场景空间（场景 + 进入规则）

> 本产品是单一扁平清单，不是意图驱动多场景系统；"场景"= 胶囊的几种用途态。任意态间可直接互跳。

| 场景ID | 场景名称 | 触发规则 | 任务类型 | 使用的流程模板 |
|--------|---------|---------|---------|-------------|
| S1 | 热键捕获 | 全局热键（默认 ⌥Space，可改）从**任意 app/任意胶囊态**触发 | 写操作 | 模板 1·捕获 |
| S2 | hover 瞄看 | 光标进入右缘热区并**驻留 150–250ms 且低速**（去抖，F3） | 只读瞥视 | 模板 2·瞄看 |
| S3 | 勾选完成 | 在展开态点某条的勾选框 | 写操作 | 模板 3·完成 |
| S4 | 编辑/排序 | 展开态点某条文字（改）/ 拖拽（排序）/ 行内"+"补一条 | 写操作 | 模板 4·编辑 |
| S5 | 空态 | 无未完成项时进入展开/静止 | 只读 | 模板 5·空态 |
| S0 | 静止常驻 | 默认态：未交互时 | 被动展示 | —（见 §3 idle） |

### 1.4 意图→场景映射表

| 意图类型 | 意图示例 | 映射到场景 | 需要澄清？ |
|---------|---------|----------|-----------|
| 记录 | "刚想到要做 X" | S1（热键，最快） | 否 |
| 查看 | "还有啥没做" | S2 / 静止数字 | 否 |
| 完成 | "这件做完了" | S3 | 否 |
| 整理 | "调下顺序 / 补两个字" | S4 | 否 |
| 误触 | 鼠标只是路过右缘 | **不进 S2**（去抖拦截，F3） | 否（设计上拦截） |

### 1.5 场景覆盖度检查

```
☑ 旅程时间线每个时段都有对应场景（捕获/瞄看/完成/编辑/空态）
☑ 每种任务类型都有场景：只读(S2/S5) / 写(S1/S3/S4)
☑ "模糊意图"在本产品不存在（无 NLP/无 Agent）；误触有拦截路径（去抖）
☑ "什么都不想做"= S0 静止常驻，零信息泄漏
```

## 2. 核心交互模型

### 2.1 意图表达方式（触发入口）

| 方式 | 描述 | 适用场景 | 优先级 |
|------|------|---------|-------|
| 全局热键 | 任意 app 之上召唤捕获（写） | S1 | **主要**（北极星路径） |
| Hover 边缘热区 | 驻留去抖后展开瞄看（看） | S2 | 辅助 |
| 单击胶囊 / 菜单栏图标 | 钉住展开；**hover 不可靠时的兜底入口（F3）** | S2/S4 | 兜底 |
| 静止数字 | 不展开即知未完成负担 | 被动 | 被动瞥视 |

### 2.2 系统响应模式

| 模式 | 触发条件 | 用户感知 |
|------|---------|---------|
| 就地形变 | 任意态切换 | 同一元素 150–250ms spring 形变（reduced-motion 下降级，见 §5） |
| 即时落库 | 回车提交一条 | 一闪微确认（toast/微动效）后缩回静止，**不停留** |
| 友好空态 | 无未完成项 | 展开显示"清零"友好引导而非空白（F-empty） |

### 2.3 流程模板集（本产品的"模板"= 胶囊用途态的交互流）

#### 模板 1 · 捕获（S1，北极星）
```
全局热键（任意态/任意 app）
  → 胶囊就地形变为带聚焦输入框的捕获面板（强制进 capture 态，夺焦点，§3）
  → 用户打字（唯一必填=标题；无清单/日期/项目选择）
  → 用户可以：
     ├─ Enter → 落入清单 → 一闪微确认 → 缩回静止 → 焦点归还原 app（§3 焦点回归）
     ├─ Shift/连续 Enter → 连记多条（可选，design-brief 定）
     └─ Esc → 放弃 → 缩回静止 → 焦点归还
```
用户权力：随时 Esc 取消、写入态只有 Enter/Esc 两个出口（无模态对话框、无必填项）。

#### 模板 2 · 瞄看（S2）
```
光标进右缘热区 + 驻留去抖通过（150–250ms 低速）
  → 胶囊形变成清单（peek 态：可点不夺焦，§3）
  → 用户扫视未完成项
  → 用户可以：
     ├─ 移开 → 自动缩回静止
     ├─ 点某条勾选框 → 升级模板 3
     ├─ 点某条文字/拖拽/行内"+" → 升级模板 4
     └─ 按热键 → 升级模板 1（强制进 capture）
```
用户权力：随时移开即收、随时升级到写/编辑。

#### 模板 3 · 完成（S3）
```
点某条勾选框
  → 愉悦勾选动效（短暂划线/淡出，给完成快感）
  → 该条进入"待清走"（撤销窗口 = N 秒，见 §3 完成态）
  → N 秒后（或下次展开时）自动移出活跃流；静止数字 -1
  → 撤销入口：N 秒内可一键还原
```
用户权力：N 秒内可撤销；超时即清，不可找回（v1 无已完成历史，jotter 定位）。

#### 模板 4 · 编辑/排序（S4）
```
点文字 → 行内可编辑 / 拖拽 → 重排（邻行缓动让位）/ 行内"+" → 原位长出可编辑新行
  → 编辑时该行临时夺焦点（peek→局部 capture，§3）
  → 失焦/Enter → 保存；Esc → 放弃本次编辑
```
用户权力：就地编辑不跳页不开模态；可取消单次编辑。

#### 模板 5 · 空态（S5）
```
无未完成项
  → 展开/静止显示友好"清零"态（如"今日清零 ✓，敲 ⌥Space 记一条"）
  → 不留空白；静止数字显示 0 或一个轻量"全清"标识
```

### 2.4 流程模板使用规则
```
☑ 每个场景标注了所用模板（S1→1, S2→2, S3→3, S4→4, S5→5）
☑ 场景内模板升级链已定义：peek(2) → 完成(3)/编辑(4)/捕获(1)
☑ 异常/边界路径覆盖：误触(去抖拦截)、hover 不可靠(兜底单击)、全屏(热键始终可用) —— 见 §5
```

## 3. 控制权流转模型 →【重定向】胶囊状态机 + 键盘焦点归属

> 这是本产品最核心的结构设计，直接落实 Oracle F2（焦点交接）+ F6（态自解释）。无 Agent，故无"字段级 Agent 控制"。

### 3.1 胶囊状态定义（态 × 可见信号 × 可用操作 × 键盘焦点）

| 状态 | 含义 | 可见信号（F6 自解释） | 可用操作 | 键盘焦点（F2） |
|------|------|--------------------|---------|--------------|
| **idle** 静止 | 常驻药丸 | 深色药丸 + 顶部清单/对勾符号(F1) + 下方未完成数 + 柔影 | 无（被动）；鼠标可穿透底层 app | **不可聚焦**，`ignoresMouseEvents=true`，不抢焦点 |
| **peek** 瞄看 | hover 展开的清单 | 药丸形变成清单面板；行 hover 高亮；**无输入光标** | 扫视、点勾选框、点文字编辑、拖拽、行内"+" | **可点但不夺 key**；仅当开始编辑某行/补一条时该控件**临时成 key**，失焦即归还 |
| **capture** 捕获 | 带聚焦输入框的写入面板 | 出现**聚焦输入框边框 + 文本光标 + 占位"记一条…"**（与 peek 肉眼可辨） | 打字、Enter 提交、Esc 取消 | **主动成 key，接管键盘**（NSPanel `.nonactivatingPanel`+`becomesKeyOnlyIfNeeded`：不激活整个 app，但输入框收键盘） |
| **confirm** 确认 | 提交后的一闪反馈 | 一闪微 toast/动效，随即缩回 | 无（瞬时） | 释放 key，准备归还原 app |

### 3.2 状态转移条件 + 焦点交接规则

| 从 | 到 | 触发 | 焦点动作 |
|----|---|------|---------|
| idle | peek | 右缘热区驻留去抖通过 | 不夺焦（保持原 app key） |
| idle/peek/任意 | capture | **全局热键（任意态强制进 capture）** | **强制接管键盘**（becomesKey），记下"来源 app"以便归还 |
| peek | capture | 点行内"+"或开始补一条 | 该输入控件临时成 key |
| capture | confirm | Enter 提交 | 提交落库 |
| capture | idle | Esc 取消 / confirm 完成 | **归还焦点给来源 app**（§3.3） |
| peek | idle | 光标移开（去抖） | 无（焦点本就在原 app） |

**关键规则（消除 F2 焦点悖论）：**
1. idle/peek 默认**绝不抢焦点**；只有"用户主动写"（capture 或行内编辑）才临时夺 key。
2. **热键在任意态触发 → 一律强制进 capture 并接管键盘**（不论当前是 idle 还是 peek，行为一致、可预测）。
3. "用户在 peek 里补一条打到一半又按了热键" → 视为进入/保持 capture，复用当前输入，不再二次形变。

### 3.3 焦点回归规则（落实 F11，分级承诺）

| 承诺级别 | 内容 | 备注 |
|---------|------|------|
| 最低（必达） | capture 结束后**回到来源 app 且不激活胶囊/不留焦点在胶囊** | non-activating panel 保证来源 app 未真正失焦 |
| 尽力（best-effort） | 回到来源 app 的**原插入点**（光标精确位置） | 可能做不到，Stage 2 spike 实测，标 best-effort |

边界场景须在 Stage 2 spike 实测：(a) 来源是文本输入位 (b) 桌面无聚焦窗 (c) 来源 app 全屏。

### 3.4 并发/边界
- 同时只有一个胶囊态；热键总是收敛到 capture（无并发歧义）。
- 多显示器：胶囊**跟随光标所在屏**的右缘；监听屏参数变化重吸附（研究 Part 2：物理坐标 ×scaleFactor 避偏移）。

## 4. 信任建设路径 →【N/A，重定向为可发现性/上手】

> 非 AI 产品，无"信任 Agent"问题（T0–T4 不适用）。本节重定向为**首次可发现性**（落实 F1 语义歧义）。

| 关键事件 | 设计目标 | 设计要点 |
|---------|---------|---------|
| 首次启动（Day 1） | 让用户知道"这是待办、怎么记" | 一次性引导气泡："这是你的待办，敲 ⌥Space 随手记一条；鼠标移到这儿看清单"。引导只出现一次。 |
| 第一次想记 | 建立热键肌肉记忆 | 引导 + 菜单栏图标兜底；静止药丸顶部用明确清单/对勾符号降低"这是什么"的歧义（F1）。 |
| 第一次想看 | 发现 hover 入口 | hover 去抖展开 + 单击兜底；静止数字本身暗示"有内容可看"。 |
| 误以为是系统指示器 | 消除 F1 歧义 | 顶部符号 = 清单/对勾（非抽象 logo）；数字语义在引导里明确为"未完成待办数"。 |

> Evaluability 设计（模板级判断时间）：N/A——无 AI 输出需判断；用户对"自己刚记的一条"天然可判断。

## 5. 系统降级策略 →【重定向为动效/全屏/hover 三类退化】

> 本产品本地、无后端服务，故无"Agent/服务降级"。重定向为三类真实退化（落实 F7 reduced-motion、F5 全屏/多屏、F3 hover 脆弱）。

### 5.1 降级级别

| 级别 | 触发条件 | 用户能做什么 | 退化表现 |
|------|---------|-----------|---------|
| L0 · 正常 | 一般情况 | 全部 | 全动效形变、hover 展开、贴边常驻 |
| L1 · 减动效 | 系统 `prefers-reduced-motion` 开启 | 全部（功能不减） | 形变降级为**淡入淡出/瞬时尺寸变化**，保留"从哪长出/缩回哪"的因果，去位移弹性；确认降级为静态 checkmark + 自动收起（F7） |
| L2 · 他人全屏 | 别的 app 进入原生全屏 | **写入(热键)始终可用**；看/管理可能不可用 | 胶囊 best-effort 浮起；**若不浮起则主动隐退**，仅保留全局热键捕获（北极星不丢）。最终承诺级 Stage 2 spike 后定（F5） |
| L3 · hover 不可靠 | 快速划过/系统繁忙致 hover 漏检 | 全部 | 去抖拦截误触；**单击胶囊/菜单栏图标作为等价兜底入口**，看列表不 100% 依赖 hover（F3） |

### 5.2 降级切换机制

| 从 | 到 | 触发 | 用户感知 | 自动/手动 |
|----|---|------|---------|----------|
| L0 | L1 | 系统开启 reduced-motion | 无打扰，动效静默降级 | 自动 |
| L0 | L2 | 检测到他人全屏 Space | 胶囊隐退/或浮起；热键仍可记 | 自动 |
| L0 | L3 | hover 漏检 | 用户改用单击/菜单栏，无报错 | 自动/用户自适应 |
| L2 | L0 | 退出他人全屏 | 胶囊重现并重吸附右缘 | 自动 |

### 5.3 降级状态体验底线
```
☑ L2（他人全屏）下，"记一条"的北极星路径（全局热键捕获）100% 可用
☑ reduced-motion 下，所有功能可用，态切换的视觉因果不丢（只是不弹/不位移）
☑ hover 不可靠时，有非 hover 等价入口（单击/菜单栏），用户不被困死
☑ 用户始终能判断胶囊当前态（§3 态信号），不会"不知道它在干嘛"
☑ 空态有友好引导，不是空白（S5）
```
