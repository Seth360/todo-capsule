# todo-capsule「看清单」大面板 · 能力集设计

> 母体已锁：深色 Granola 药丸（`#1E1E20`）/ 扁平无日期单桶 / 双入口（⌥Space 热键写 · hover peek 看）/ 勾掉即清 / Inter + accent `#34C75A` / morph 180–220ms spring。
> 本文档定义的是**主动打开的大面板**（点击召唤、持久驻留、只有点关闭才收）——它是 hover-peek（~320px 一瞥）之上的**第三态深度**，不是另一个 app。原则：**宁可少而精**，跨 8 app 反复验证 + 真正契合「持久面板」者才留。

---

## 1. 该留的能力（KEEP，共 7 条，按价值排序）

### K1 · 聚焦输入框 + 实时 NL 解析 token（面板内顶部常驻）
- **来自**：Todoist（一行解析 + 实时染色 token）、TickTick（Smart Recognition 高亮可点改）、Things 3（Jump Start）、Quick Add（全员共识）。
- **为什么**：研究已认定「全局热键召唤捕获」是母体灵魂；大面板打开后，输入框就该**已获焦、光标已闪**，打字即记、回车即落、连续记多条不离手。这是面板存在的第一理由——把「记」和「看」放进同一个持久面。
- **轻量化**：面板顶部一条常驻输入框（非弹模态）。**MVP 严守纯扁平无日期**，所以这里**只解析标题**，不抄 Todoist 的 `#项目/@标签/p1/日期` 全套染色——保留输入框 + 实时高亮的「视觉骨架」，但 v1 高亮目标为空（为未来留接口、当前不暴露）。回车落到顶部，输入框清空保持焦点。

### K2 · 完成项「划线 + 4s 撤销 + 沉底/移出」闭环
- **来自**：Todoist（划除 + 5s undo 浮条）、Bear/Apple Notes（完成自动沉底）、Things 3（延时滑入 Logbook 留反悔窗口）、Apple Reminders（填色 + 上滑离场）。
- **为什么**：这是母体已锁的 D-006「勾掉即清 + ~4s 撤销」的**面板放大版**。小胶囊里勾完直接走；大面板因为停留更久、一次处理多条，**需要一个可见的撤销条 + 完成项短暂沉底**，给批量勾选时的安全感和「我清掉了一批」的成就闭环。
- **轻量化**：勾选 → 划线 + 行下沉到「今日已完成」轻分隔下方（灰、划线）→ 底部滑入一条「已完成 · 撤销」浮条（4s）→ 超时移出活跃流，静止计数 −1。**v1 不做可翻阅 Logbook**（Things/Todoist 的历史库一律砍），沉底只是离场前的缓冲。

### K3 · 静则隐 / 动则现的悬停操作（hover 才出复选框与控件）
- **来自**：Superlist（悬停才出复选框，静止像纯笔记）、Bear（左边距拖拽圆点/折叠 chevron hover 才浮现）、Notion（2025 内联控件 hover 才显形）。
- **为什么**：持久面板长时间在眼前，**静止态必须极干净**才不打扰（母体第一约束）。把复选框、行内编辑、删除、拖拽手柄全部收进 hover 态，平时每行只有一行文字，信息密度逼近纯清单。
- **轻量化**：行默认 = 纯标题（13px `#F2F2F4`）。行 hover → 左侧淡入圆形复选框（120ms opacity）+ 右侧淡入「⋯」轻操作（删除/置顶）。不做 Bear 的拖拽圆点常驻——拖拽手柄也藏在 hover。

### K4 · 圆形勾选 + 物理感微动效（spring，非线性）
- **来自**：Things 3（圆圈→打勾→延时滑出，ADA 级动效即反馈）、Superlist（弹簧物理手感）、Apple Notes（打勾弹性填充 + 划线渐入）、TickTick（drag-and-release tick）。
- **为什么**：母体五大约束含「动效好」，且圆形勾选圈是这一品类的**统一视觉母题**。大面板里勾选是最高频动作，必须有满足感的物理反馈。
- **轻量化**：单一圆形 checkbox（accent `#34C75A` 填充）；点击 = 填色 + 内描勾 + 该行 spring 轻微下沉。复用母体已定的 180–220ms spring，**不引入 Lottie**（自绘 SwiftUI 即可，避免依赖）。

### K5 · 行内即时编辑（双击/单击进入，回车提交，无弹窗）
- **来自**：Notion（内联就地编辑，零上下文切换）、Superlist（行内连续建任务 + Markdown 即时转）、Bear（所见即所得 inline 格式化）。代码侧已实现双击行内编辑。
- **为什么**：大面板的核心是「在原地处理清单」。改一条措辞、补一条、删一条都不该跳页或开抽屉——否则它就退化成需要管理的 app，违背「轻量」。
- **轻量化**：双击行 → 当场变可编辑文本，回车提交 / Esc 取消。**不抄 Superlist 的 `/` 斜杠菜单和 Markdown block 模型**（那是富文本编辑器，过重）；纯单行纯文本编辑即可。

### K6 · 拖拽排序 + 单层「置顶/pin」
- **来自**：Bear（左边距拖拽圆点重排 + Pinned 置顶区）、Notion（拖拽改归属）、Apple Notes（拖拽小圆点重排，实时让位）。代码侧已实现 `List.onMove` 拖拽排序。
- **为什么**：纯扁平清单变长后，唯一不增加结构复杂度的「整理」手段就是**手动排序 + 把一两条关键的钉在顶部**。比标签/分组/优先级四档都轻，却覆盖 90% 的「这条最要紧」诉求。
- **轻量化**：拖拽手柄藏在 hover（见 K3），拖动时其它行 spring 让位 + 被拖行轻微抬起阴影。pin = hover「⋯」里一个动作，钉住的行带一个极小 accent 圆点常驻顶部。**不做** Bear 的本地/全局双档置顶，只一档。

### K7 · 友好「清零」空态 + 引导
- **来自**：MS To Do（My Day 每日干净起点）、Things 3 / 母体已锁 D-009「今日清零 ✓」。
- **为什么**：持久面板每天都会被看到，空的时候不能是冷白板。一个温暖的「清零」态既是情绪奖励，又是教学位（提示热键、提示 hover）。
- **轻量化**：无未完成项 → 面板中部显示「今日清零 ✓ · ⌥Space 记一条」，首启一次性带 hover 提示。直接复用母体已定文案与 token，零额外设计。

> **被认真考虑但落选的边界候选**（说明克制取舍）：
> - MS To Do 的 **My Day 每晚归零 + Suggestions pull 回流**——机制极优，但需要「全部待办池 vs 今日承诺」两个层，而母体是**纯单桶**，引入即破坏扁平。**落为 v2 候选，v1 不做**。
> - Apple 的 **今天按上午/下午/晚上分时段**——优雅，但依赖「时间」维度，母体已锁**无日期**。**v1 不做**。
> - Sunsama 的 **工作量进度条 / 未完成结转卡**——需要时间估算或日期，与无日期母体冲突。**v1 不做**。

## 2. 明确砍掉（CUT）

| 砍掉的能力 | 来自 | 砍的原因 |
|---|---|---|
| **项目 / Area / Folder / Section 多级组织** | Things 3, Todoist, TickTick, Notion | 母体已锁纯扁平单桶。两级组织把胶囊变成 GTD 项目经理，头号失败模式（scope creep）。 |
| **多视图切换（看板 / 日历 / 时间线 / 画廊 / 矩阵）** | Todoist, TickTick, Notion, Obsidian | 持久面板只需「一个清单视图」。多视图是 PM 工具堆料，与极简持久面板直接冲突。 |
| **日期 / 截止 / 提醒 / 重复规则** | 全员 | 母体已锁无日期。重复规则解析是大坑；提醒是另一套系统权限。v1 一律不做。 |
| **四档优先级 / 标签系统 / 旗标三套并行标记** | Todoist, TickTick, Apple | 标记体系冗余。大面板只保留「pin 置顶」一个一键标记维度（K6），其余砍。 |
| **自定义过滤器 / 查询 DSL / 智能列表规则编辑器** | TickTick, Todoist, Obsidian, Apple | 规则构建器=能力换复杂度。单桶清单无需过滤；最多一个搜索框（见 §3 备注，非 KEEP）。 |
| **协作 / 共享 / 指派 / 评论 / Workspaces** | Todoist, Superlist, Notion, MS, Sunsama | 单机个人胶囊，协作维度正交，整块砍。 |
| **番茄钟 / 习惯追踪 / 时间估算计时 / 统计 / Karma** | TickTick, Sunsama | 另一个产品域；Karma 经证实是反向激励虚荣游戏，与「不打扰」直接冲突。 |
| **富媒体（图片 / 附件 / 表格 / 扫描 / 富文本 block）** | Superlist, Bear, Notion, Apple | 待办行最多带「一行纯文本」，不做富文本编辑器。存储与渲染成本高、偏离「看清单」。 |
| **多层嵌套子任务 / 无限层级** | 全员 | v1 连一层子任务都暂不做（保持每行一条）；嵌套是项目管理膨胀。 |
| **AI 拆解 / 自动排程 / 会议记录 / 语音转任务** | Todoist, Superlist, Sunsama, Apple | 研究已判 not_suitable（纯 GUI 产品）；引入即破坏轻量与本地优先。 |
| **多源 Inbox 聚合 / 第三方集成 / 邮件转任务 / 日历双向同步** | Akiflow, Amie, Notion, MS | 「生产力中枢」定位，与本地常驻胶囊相悖。只做本地清单。 |
| **可翻阅的历史库（Logbook / Completed 浏览）** | Things 3, Todoist, Bear | 完成项沉底缓冲即可（K2），不做可翻阅历史视图。 |
| **多主题 / 背景图 / 白噪音 / 自定义图标徽章** | TickTick, Superlist | 一套精致深色默认即可，定制是噪音。 |

## 3. 大面板信息架构与交互

### 3.1 面板分区（自上而下，单栏）
```
┌─────────────────────────────┐  ← 面板 #232326 / 圆角14px / 描边 rgba(255,255,255,.08)
│ ▸ 顶栏：标题"今天" · 计数 · [✕]│  L3 引导/计数 + 关闭按钮（右上）
├─────────────────────────────┤
│ ⌥Space  [ 记一条…           ]│  ← K1 常驻聚焦输入框（打开即获焦）
├─────────────────────────────┤
│ ○ 待办行（hover 出复选/⋯/拖手）│  ← K3/K4/K5/K6 活跃流（pin 项在最上，带 accent 点）
│ ○ 待办行                      │
│ ○ 待办行                      │
│   … 内滚（max-height min(70vh)）│
├─────────────────────────────┤  ← 轻分隔线（仅在有已完成项时出现）
│ ⊘ 已完成（划线·灰·短暂沉底）   │  ← K2 离场缓冲区
└─────────────────────────────┘
        [ 已完成 · 撤销 ]          ← K2 撤销浮条（4s，底部滑入）
```
- 宽度 ~360–400px（比 peek 的 320px 略宽，因常驻输入框 + 操作控件需要呼吸）；高度 max-height `min(70vh, 内容)` 内滚。
- 空态（K7）：活跃流区替换为居中「今日清零 ✓ · ⌥Space 记一条」。

### 3.2 与小胶囊的分工（三态深度，职责不重叠）
| 形态 | 触发 | 职责 | 是否可写 | 是否持久 |
|---|---|---|---|---|
| **idle 药丸** | 默认 | 一眼看未完成数（零文字泄漏） | 否 | 是（边缘常驻） |
| **peek 一瞥**（已有） | hover 驻留 150–250ms | 快速扫今日有哪些（行 hover 高亮，无输入光标） | 否（只看） | 否（移开自动收） |
| **大面板**（本文档） | **单击药丸 / 菜单栏图标 / 热键再次召唤** | 深度处理：连续记、勾、编辑、排序、看全 | **是** | **是（只有点 ✕ 或 Esc 才收）** |
| **热键 capture**（已有） | ⌥Space | 极速记一条即走 | 是（一次性） | 否（confirm 一闪即收） |

- **关键边界**：peek 是「被动一瞥、自动收」，大面板是「主动打开、不自动收」。这是 hover 脆弱性的兜底——D-002 已定「单击药丸 = 等价兜底入口」，大面板正是这个兜底的归宿，看清单不 100% 依赖 hover。
- 热键 ⌥Space 默认仍走极速 capture（母体不变）；**当大面板已打开时**，⌥Space = 聚焦面板内输入框（不另弹 capture），避免双重捕获面。

### 3.3 打开 / 关闭交互（含关闭按钮）
- **打开**：单击 idle 药丸 → 药丸 spring 形变长成大面板（灵动岛同源形变，180–220ms spring，从药丸所在右缘锚点长出）；输入框**立即获焦**、光标闪烁。
- **关闭**（三条等价路径，都明确）：
  1. **右上角 ✕**（hover 才加深，静止态轻 `#9B9BA1`）——满足「只有点关闭才收」的字面承诺；
  2. **Esc**——键盘流离场（若输入框有内容，第一次 Esc 先清空输入、第二次才关面板）;
  3. **点击面板外**——可选关闭（**默认开**，符合「不打扰、随手点开点关」；但因 non-activating 写入特性，需 spike 验证点外关闭不误伤；若不稳则降级为「仅 ✕/Esc 关」）。
- **关闭动效**：面板 spring 缩回药丸（缩回到原锚点，保留「缩回哪」的因果），计数同步更新。
- **绝不**：不做最小化、不做内部侧栏折叠（母体本身就是点开才在），不做 Slim Mode（CUT 已列）。

### 3.4 键盘流（Type Travel 精神，面板获焦即可全键盘）
- 打开即焦点在输入框：**直接打字即记**（抄 Things 3 Type Travel / TickTick 即输即记）。
- `Enter` 落一条、输入框清空保持焦点（连续记，抄 Superlist 行内连续建）；
- `↑/↓` 在行间移动焦点（输入框为空时按 ↓ 进入列表）；
- 焦点在某行：`Space`/`⌘K` 勾选完成、`Enter` 进入行内编辑、`⌘⌫` 删除、`⌘↑/⌘↓` 拖动排序、`⌘P` 置顶；
- `⌘Z` 撤销刚才的完成/删除（呼应 4s 撤销条）；
- `Esc` 两段式离场（见上）。

> **搜索/过滤**：长清单时**可选**保留一个极轻的「即输即筛」——但**不暴露查询语法**（CUT 已砍 DSL）。鉴于母体是单桶且 v1 清单预期不长，**搜索列为 v2 候选，v1 不做**（避免顶栏再加一个常驻控件）。

## 4. UI 与动效主张

### 4.1 视觉（深色 Granola 体系，复用已锁 token，零新增色板）
- **面**：面板 `#232326` / 描边 `rgba(255,255,255,.08)` / 柔影 `0 8px 24px rgba(0,0,0,.35)` / 圆角 **14px**。
- **字（Inter）**：待办 13px/400 `#F2F2F4`；计数 12px/600 accent；次要（顶栏标题/引导）12px/400 `#9B9BA1`；占位 13px `#6E6E74`。
- **accent**：`#34C75A`（呼应 Granola 绿；勾选填充、计数、pin 圆点统一用它——抄 Things 3「圆圈作为统一视觉母题」）。
- **密度**（抄 Attio 三层 + Notion 2025 降噪）：行高 ~32px、8pt 栅格、面板内距 12–16px。静止态每行只一行文字 + 圆 checkbox 轮廓，**留白克制、单栏窄列、行高舒展**（Todoist「清单呼吸感」）。已完成项 = 灰 `#6E6E74` + 划线。
- **怎么好看的关键**：① 默认极安静（hover 才出控件，K3）；② 唯一强色是 accent 绿，作为勾选/pin/计数的统一点缀（颜色即层级，抄 Todoist 旗标语言）；③ 深色面板长清单对比度需在原型实测调校（design-brief 已记 deferred）。

### 4.2 动效（全部服务于「状态变化的可理解性」，复用母体 spring，附 reduced-motion 双规格）
| 动作 | 默认动效 | 时长 / 缓动 | reduced-motion 降级 |
|---|---|---|---|
| **打开**（药丸→面板） | 灵动岛同源 spring 形变，从右缘锚点长出，内容淡入 | 180–220ms spring | 120ms 透明度+尺寸渐变，保留「从哪长出」因果 |
| **关闭**（面板→药丸） | spring 缩回原锚点 | 180–220ms spring | 120ms 渐变 |
| **新增一行**（回车落条） | 新行从输入框处轻微下滑入位 + 既有行 spring 让位 | 160ms ease-out / spring | 100ms 透明度渐入 |
| **勾选完成**（K4） | 圆圈填 accent + 内描勾 + 该行轻微下沉 | 150ms spring | 静态填色 + 划线，无位移 |
| **完成沉底**（K2） | 划线 + 平滑滑到「已完成」分隔下方并淡灰 | 200ms ease-in-out | 直接置灰沉底，无滑动 |
| **撤销浮条** | 底部弹性滑入、4s 后滑出 | 入 200ms / 出 160ms | 透明度淡入淡出 |
| **删除一行** | 行向右淡出 + 下方行 spring 上移补位 | 160ms | 透明度淡出 |
| **拖拽排序**（K6） | 拖行抬起阴影、其它行实时让位（物理跟手） | 跟手 / 落位 spring | 跟手 / 落位无弹性 |
| **hover 出控件**（K3） | 复选框/⋯/拖手柄 opacity 0→1 | 120ms ease | 同（纯透明度，本就无位移） |

- **动效哲学**（抄 Superlist「最好的动效用户几乎察觉不到」+ Things 3「动效即反馈、不炫技」）：弹簧物理而非线性缓动；克制、有实体感、绝不喧宾夺主。母体已锁「morph 180–220ms spring / reduced-motion 120ms 渐变」，本面板所有形变**严格沿用同一套参数**，保证 idle↔peek↔面板三态形变手感统一（同一个元素在长大/缩小，不是不同窗口）。

## 5. 与现有极简 MVP 的关系

**核心原则：小胶囊（idle / peek / 热键 capture）一行不动，全部新增能力都关进「大面板专属」深度。** 母体的简单/不打扰由前三态守住；大面板是用户**主动选择进入**的深度态，深度不会泄漏到日常视野。

| 能力 | 归属 | 说明 |
|---|---|---|

**给 Stage 2（原生 SwiftUI）的落地提示**：大面板复用 idle↔peek 已有的 `MorphContainer` spring 参数，是「同一形变元素的第三档尺寸」，不是新窗口/新 NSPanel——避免再开一层焦点交接。non-activating 写入 + 点击面板外关闭的兼容性，并入 PROGRESS 里已挂的 NSPanel spike 一起真机验证。
